Windows DFIR Utility

RDP Cache Review Workbench

Search, copy, extract, and reconstruct Windows Remote Desktop bitmap cache artefacts using a repeatable PowerShell workflow with pinned, locally cached helper tools.

Download v0.2.0 Read documentation

Purpose

Windows Remote Desktop can leave bitmap cache files on the client machine. Those files may contain visual fragments from previous RDP sessions. This project packages a defensive workflow for finding those files, preserving a working copy, extracting bitmap tiles, and attempting automated reconstruction.

Current release

Version v0.2.0 includes the direct PowerShell script, the importable RdpCacheWorkbench module, a GitHub release ZIP, CI validation, package checks, and optional Authenticode signing support for release maintainers. 

Import-Module .\RdpCacheWorkbench\RdpCacheWorkbench.psd1
Invoke-RdpCacheReview

The module is structured for future PowerShell Gallery publication. Until then, download the release ZIP or clone the repository and import the module locally.

Workflow

1

Find cache files

Search fixed drives or a user-supplied folder for Cache????.bin and bcache*.bmc.

2

Preserve a copy

Copy files into a working case folder and generate a SHA-256 manifest.

3

Extract tiles

Use the locally cached pinned ANSSI bmc-tools release to extract BMP tile fragments from the cache files.

4

Reconstruct

Run the locally cached pinned BriMor Labs rdpieces release against the extracted BMP tiles to generate reconstruction candidates.

Quick start

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\Invoke-RdpCacheReview.ps1

Or run the module command after importing the module:

Import-Module .\RdpCacheWorkbench\RdpCacheWorkbench.psd1
Invoke-RdpCacheReview

For a targeted run:

.\Invoke-RdpCacheReview.ps1 -SearchRoot "$env:LOCALAPPDATA\Microsoft\Terminal Server Client\Cache"

Use a custom fixed tool cache:

.\Invoke-RdpCacheReview.ps1 -ToolCacheRoot D:\RdpCacheWorkbench\tools

Operational note

Treat extracted tiles and reconstructed images as sensitive material. Output can contain fragments of emails, documents, applications, credentials, or personal data. Use only on systems where you have explicit authorisation.

Pinned third-party tooling

This release pins ANSSI bmc-tools to commit 5a4cad32be78b3b874aeec910cb478e04ba3501e and BriMor Labs RDPieces to commit 2a74aeb4b8f42fac1af1f6c9d721fcb299224021. The script no longer pulls moving master.zip branch archives.

Pinned helper tools are cached under %LOCALAPPDATA%\RdpCacheWorkbench\tools by default, or under a user-supplied -ToolCacheRoot. After first successful setup, later runs can reuse those tools offline.

RDPieces is run through a hardened wrapper that stages input under a temporary subst drive to reduce command-line path injection and path parsing risk.
